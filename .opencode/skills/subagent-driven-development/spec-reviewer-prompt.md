# 사양 적합성 검토자 프롬프트 템플릿

사양 적합성 검토자 서브에이전트를 보낼 때 이 템플릿을 사용한다.

**목적:** 구현이 사양과 일치하는지 검증한다.

```
Task tool (general-purpose):
  description: "Review spec compliance for Task N"
  prompt: |
    당신은 구현이 사양과 일치하는지 검토한다.

    ## 요청 내용

    [작업 요구사항 전문]

    ## 구현자가 만들었다고 주장하는 내용

    [구현자 보고서에서 가져온 내용]

    ## 중요: 보고서를 그대로 믿지 않는다

    실제 코드를 읽고 요구사항과 한 줄씩 비교한다.
    `src/routes/**` 테스트는 사용자 요구가 있었는지 확인한다.

    보고:
    - ✅ 사양 적합
    - ❌ 발견한 문제: [누락 또는 과잉 작업을 파일:행 번호와 함께 적는다]
```
