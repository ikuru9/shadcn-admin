# 구현자 서브에이전트 프롬프트 템플릿

구현자 서브에이전트를 보낼 때 이 템플릿을 사용한다.

```
Task tool (general-purpose):
  description: "Implement Task N: [task name]"
  prompt: |
    당신은 Task N: [task name]을 구현한다.

    ## 작업 설명

    [계획의 작업 전문을 여기에 붙여 넣는다. 서브에이전트가 파일을 읽게 하지 않는다.]

    ## 맥락

    [이 작업이 어디에 속하는지, 의존성은 무엇인지, 구조적 맥락은 무엇인지 적는다.]

    ## 시작하기 전에

    요구사항이나 접근 방식에 질문이 있으면 지금 묻는다.

    ## 해야 할 일

    작업이 요구하는 것만 정확히 구현하고, 필요하면 테스트를 작성하며, 구현을 검증하고, 커밋하고, 자기 검토 후 보고한다.
    `src/routes/**`는 기본적으로 테스트 파일을 만들지 않는다. 사용자가 명시적으로 요구한 경우에만 작성한다.

    ## 보고 형식

    - 상태: DONE | DONE_WITH_CONCERNS | BLOCKED | NEEDS_CONTEXT
    - 구현한 내용
    - 수행한 검증과 결과
    - 변경한 파일
    - 자기 검토 결과
    - 문제 또는 우려 사항
```
