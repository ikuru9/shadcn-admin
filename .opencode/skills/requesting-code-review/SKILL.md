# 코드 검토 요청하기

문제가 커지기 전에 코드 리뷰 서브에이전트를 보내 검토한다.

## 검토 시점

- `subagent-driven-development`의 각 작업이 끝날 때
- 큰 기능을 마쳤을 때
- 병합 전에

## 요청 방법

1. git SHA를 구한다.
2. 정확한 작업 또는 계획 맥락과 함께 `code-reviewer`를 보낸다.
3. 피드백을 반영한다.

## 주의 사항

- 단순하다고 검토를 건너뛰지 않는다.
- 치명적인 문제를 무시하지 않는다.
- 중요한 문제를 고치지 않고 다음 단계로 넘어가지 않는다.
