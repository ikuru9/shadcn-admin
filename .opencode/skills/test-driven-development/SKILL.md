# 테스트 주도 개발

## 핵심 규칙

테스트를 먼저 작성한다. 실패하는 것을 확인한다. 최소 코드로 통과시킨다.

**실패하는 테스트 없이 프로덕션 코드를 작성하지 않는다.**

## 순환

1. RED - 하나의 동작에 대한 최소 테스트를 작성한다.
2. 예상한 이유로 실패하는지 확인한다.
3. GREEN - 통과할 최소 구현을 작성한다.
4. 통과하는지 확인한다.
5. REFACTOR - 통과 상태를 유지하면서 정리한다.

## 파일 기준

- 신규 기능과 수정 작업은 대응하는 단위 테스트 파일을 함께 둔다.
- 단, `src/routes/**`는 사용자가 명시적으로 요구한 경우에만 테스트 파일을 둔다.
- 버그 수정은 재현 테스트 파일이 먼저 있어야 한다.

## 최종 규칙

프로덕션 코드는 반드시 먼저 실패한 테스트가 있어야 한다.
