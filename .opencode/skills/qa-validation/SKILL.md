# QA Validation

## 목적
보이는 화면과 연결된 데이터가 실제로 일치하는지 확인한다.

## 작업 방식
- 관련 파일을 최소 2개 이상 교차로 읽는다.
- 화면, 훅, 스키마, API 응답의 연결을 본다.
- 문제를 재현 가능한 문장으로 남긴다.

## 체크포인트
- 빈 상태, 에러 상태, 로딩 상태를 빠뜨리지 않는지 확인한다.
- 수정 요청은 사실 기반으로 짧게 적는다.
