# 기능 구현 계획 템플릿

**목표:** 한 문장으로 기능의 결과를 설명한다.

**구조:** 어떤 파일과 경계를 어떻게 나눌지 설명한다.

**기술 스택:** 관련 기술만 적는다.

**테스트 정책:** `src/routes/**`가 아니면 대응하는 단위 테스트 파일을 함께 적는다. `src/routes/**`는 사용자가 명시적으로 요구한 경우에만 테스트 파일을 적는다.

---

### 작업 1: 준비

**파일:**
- 수정할 파일 경로를 적는다.
- 신규 또는 수정 대상에 대응하는 단위 테스트 파일 경로를 적는다. 단, `src/routes/**`는 예외다.

- [ ] 실패하는 테스트를 먼저 작성한다.
- [ ] 테스트가 실패하는지 확인한다.
- [ ] 최소 구현을 작성한다.
- [ ] 테스트가 통과하는지 확인한다.

**단위 테스트 파일:**
- `path/to/new-or-updated.test.tsx`
