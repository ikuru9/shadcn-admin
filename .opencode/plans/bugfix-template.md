# 버그 수정 계획 템플릿

**목표:** 어떤 버그를 고치는지 한 문장으로 적는다.

**구조:** 재현, 수정, 검증 순서를 적는다.

**기술 스택:** 관련 기술만 적는다.

**테스트 정책:** 버그가 `src/routes/**` 안에 있으면 사용자가 명시적으로 요구한 경우에만 테스트 파일을 적는다. 그 외에는 재현용 단위 테스트 파일을 반드시 적는다.

---

### 작업 1: 재현

**파일:**
- 관련 파일 경로를 적는다.
- 버그 재현용 단위 테스트 파일 경로를 적는다. 단, `src/routes/**`는 예외다.

- [ ] 버그를 재현하는 실패 테스트를 작성한다.
- [ ] 실패를 확인한다.
- [ ] 최소 수정으로 문제를 고친다.
- [ ] 테스트가 통과하는지 확인한다.

**단위 테스트 파일:**
- `path/to/bug-regression.test.tsx`
