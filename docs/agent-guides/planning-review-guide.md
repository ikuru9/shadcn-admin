# Planning Review Guide

When: Read when writing specs or plans, and before sign-off or review.

- 계획 문서는 `docs/report/plans/`에 저장한다.
- 설계 문서는 `docs/report/specs/`에 저장한다.
- 계획은 파일 단위로 나누고, 정확한 경로와 검증 명령을 적는다.
- `src/routes/**`가 아니면 테스트 파일 경로를 함께 적는다.
- `src/routes/**`는 사용자가 명시적으로 요구한 경우에만 테스트 파일을 적는다.
- 사양 검토 때는 요구사항 충족, 범위 초과, 생성 코드 직접 수정 여부를 확인한다.
- 품질 검토 때는 가독성, 중복, 검증, 접근성, 정책 일치를 확인한다.
- 완료 기준에는 검증 통과, 문서 갱신, 남은 위험 정리가 포함된다.
