# TypeScript Patterns

## When to read

- When designing types or writing generics
- When you need safe type narrowing

## Rules

- Write code for strict mode
- Keep `noUnusedLocals` and `noUnusedParameters` in mind
- Prefer `type` over `interface`
- Use generics actively
- Use discriminated unions
- Use type guard functions
- Do not use `any` in production code
- Use `unknown` when needed and narrow before use
- Prefer named exports over default exports
