# Prohibitions

## When to read

- When checking what must never be done
- When reviewing mandatory compliance rules

## Prohibitions

- Do not use `npm` or `yarn`
- Do not use `@ts-ignore` or `as any` in production code
- Do not bypass git hooks
- Do not create git commits without explicit user request
- Do not make direct visual style changes yourself
- Do not use relative import paths

## Mandatory Rules

- Use `pnpm` as the only package manager
- Use the `@/*` path alias
- Check diagnostics for changed files before completion
